# Input types

    - Comfy dtype: MODEL
    - Python dtype: torch.nn.Module
- clip

    - Comfy dtype: CLIP
    - Python dtype: torch.nn.Module
- token_normalization
    - Defines the method for token normalization, affecting how text input is processed and encoded. It influences the adaptability of encoding for different text lengths and complexities.

- weight_interpretation
    - Determines how weights are interpreted during the encoding process, affecting the quality and relevance of the final encoded output.
    - Comfy dtype: COMBO[STRING]
    - Python dtype: str
- wildcard_text
    - The text input to be encoded. This parameter is the core of the node's operation, directly influencing the output.
    - Comfy dtype: STRING
    - Python dtype: str
- populated_text
    - The result of filling wildcard text, automatically generated based on the encoding process. It reflects an enhanced version of the input text.
    - Comfy dtype: STRING
    - Python dtype: str
- mode
    - Controls whether text is dynamically filled or fixed, affecting how wildcard text is processed and encoded.
    - Comfy dtype: BOOLEAN

    - Allows selecting a LoRA to add to the text, enhancing encoding with specific features or effects.
    - Comfy dtype: COMBO[STRING]

- Select to add Wildcard
    - Allows adding predefined wildcard text to further customize the encoding process.
    - Comfy dtype: COMBO[STRING]

- seed

    - Comfy dtype: INT
