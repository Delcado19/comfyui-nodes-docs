# Documentation
- Class name: WAS_Dictionary_to_Text
- Category: WAS Suite/Text
- Output node: False
- Repo Ref: https://github.com/WASasquatch/was-node-suite-comfyui

The WAS_Dictionary_to_Text node converts a given dictionary into a text representation. It plays a key role in transforming structured data into a readable format, which is essential for various applications such as data analysis, reporting, and user interfaces. This node simplifies the process of generating human-readable text from complex dictionary structures, enhancing the overall data presentation workflow.

# Input types
## Required
- dictionary
    - The 'dictionary' parameter is critical to the node's operation, as it is the primary input that the node processes to generate text. It is a structured representation of data that the node will interpret and convert into a text format. The quality of the output text heavily depends on the structure and content of the input dictionary.
    - Comfy dtype: DICT
    - Python dtype: Dict[str, Any]

# Output types
- text
    - The 'text' output is the result of the node's conversion process, which takes the input dictionary and generates a textual representation. This output is important because it provides an easily understandable, human-readable format that can be used in various contexts such as documentation, communication, or further data processing.
    - Comfy dtype: TEXT_TYPE
    - Python dtype: str

# Usage tips
- Infra type: CPU
