# Documentation
- Class name: KfDebug_Latent
- Category: debugging
- Output node: True
- Repo Ref: https://github.com/dmarx/ComfyUI-Keyframed

The KfDebug_Latent node aims to provide insight into the latent space representation generated by keyframe-based models. It serves as a diagnostic tool, allowing inspection and analysis of latent features, which is critical for understanding the underlying structure and quality of the model's data processing capabilities.

# Input types
## Required
- input_data
    - The input_data parameter is the raw data fed into the node, expected in the form of a PyTorch tensor. It is critical, as it forms the basis for the node's latent space analysis. The quality and structure of input_data directly affects the node's ability to generate meaningful latent representations.
    - Comfy dtype: torch.Tensor
    - Python dtype: torch.Tensor

# Output types
- latent_representation
    - The latent_representation output provides the processed latent space data derived from the node's analysis. It is important, as it contains the information extracted from input_data, offering a condensed view of latent features. This output is useful for further research or visualization of the model's latent space.
    - Comfy dtype: torch.Tensor
    - Python dtype: torch.Tensor

- Infra type: CPU
