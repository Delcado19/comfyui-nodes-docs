# Documentation
- Class name: AILab_QwenVL_Advanced
- Display name: QwenVL (Advanced)
- Category: 🧪AILab/QwenVL
- Output node: False
- Repo Ref: https://github.com/1038lab/ComfyUI-QwenVL

Runs a Qwen-VL vision-language model to analyze an image or video input and return a text response. Supports multiple quantization levels, attention backends, beam search, and frame sampling for video.

# Input types
## Required
- model_name
    - Pick the Qwen-VL checkpoint. First run downloads weights into models/LLM/Qwen-VL, so leave disk space.
    - Comfy dtype: COMBO
    - Default: `"Qwen3-VL-2B-Instruct"`
    - Options: `Qwen3-VL-2B-Instruct`, `Qwen3-VL-2B-Thinking`, `Qwen3-VL-2B-Instruct-FP8`, `Qwen3-VL-2B-Thinking-FP8`, `Qwen3-VL-4B-Instruct`, `Qwen3-VL-4B-Thinking`, `Qwen3-VL-4B-Instruct-FP8`, `Qwen3-VL-4B-Thinking-FP8`, `Qwen3-VL-8B-Instruct`, `Qwen3-VL-8B-Thinking`, `Qwen3-VL-8B-Instruct-FP8`, `Qwen3-VL-8B-Thinking-FP8`, `Qwen3-VL-32B-Instruct`, `Qwen3-VL-32B-Thinking`, `Qwen3-VL-32B-Instruct-FP8`, `Qwen3-VL-32B-Thinking-FP8`, `Qwen2.5-VL-3B-Instruct`, `Qwen2.5-VL-7B-Instruct`
- quantization
    - Precision vs VRAM. FP16 gives the best quality if memory allows; 8-bit suits 8–16 GB GPUs; 4-bit fits 6 GB or lower but is slower.
    - Comfy dtype: COMBO
    - Default: `"None (FP16)"`
    - Options: `4-bit (VRAM-friendly)`, `8-bit (Balanced)`, `None (FP16)`
- attention_mode
    - auto tries flash-attn v2 when installed and falls back to SDPA. Only override when debugging attention backends.
    - Comfy dtype: COMBO
    - Default: `"auto"`
    - Options: `auto`, `sage`, `flash_attention_2`, `sdpa`
- use_torch_compile
    - Enable torch.compile('reduce-overhead') on supported CUDA/Torch 2.1+ builds for extra throughput after the first compile.
    - Comfy dtype: BOOLEAN
    - Default: `false`
- device
    - Choose where to run the model: auto, cpu, mps, or cuda:x for multi-GPU systems.
    - Comfy dtype: COMBO
    - Default: `"auto"`
    - Options: `auto`, `cpu`, `mps`, `cuda:0`
- preset_prompt
    - Built-in instruction describing how Qwen-VL should analyze the media input.
    - Comfy dtype: COMBO
    - Default: `"🖼️ Detailed Description"`
    - Options: `🖼️ Tags`, `🖼️ Simple Description`, `🖼️ Detailed Description`, `🖼️ Ultra Detailed Description`, `🎬 Cinematic Description`, `🖼️ Detailed Analysis`, `📹 Video Summary`, `📖 Short Story`, `🪄 Prompt Refine & Expand`
- custom_prompt
    - Optional override—when filled it completely replaces the preset template.
    - Comfy dtype: STRING
    - Default: `""`
- max_tokens
    - Maximum number of new tokens to decode. Larger values yield longer answers but consume more time and memory.
    - Comfy dtype: INT
    - Default: `512`
- temperature
    - Sampling randomness when num_beams == 1. 0.2–0.4 is focused, 0.7+ is creative.
    - Comfy dtype: FLOAT
    - Default: `0.6`
- top_p
    - Nucleus sampling cutoff when num_beams == 1. Lower values keep only top tokens; 0.9–0.95 allows more variety.
    - Comfy dtype: FLOAT
    - Default: `0.9`
- num_beams
    - Beam-search width. Values >1 disable temperature/top_p and trade speed for more stable answers.
    - Comfy dtype: INT
    - Default: `1`
- repetition_penalty
    - Values >1 (e.g., 1.1–1.3) penalize repeated phrases; 1.0 leaves logits untouched.
    - Comfy dtype: FLOAT
    - Default: `1.2`
- frame_count
    - Number of frames extracted from video inputs before prompting Qwen-VL. More frames provide context but cost time.
    - Comfy dtype: INT
    - Default: `16`
- keep_model_loaded
    - Keeps the model resident in VRAM/RAM after the run so the next prompt skips loading.
    - Comfy dtype: BOOLEAN
    - Default: `true`
- seed
    - Seed controlling sampling and frame picking; reuse it to reproduce results.
    - Comfy dtype: INT
    - Default: `1`
## Optional
- image
    - An image tensor to be analyzed by the model.
    - Comfy dtype: IMAGE
- video
    - A video tensor from which frames are extracted and analyzed by the model.
    - Comfy dtype: IMAGE

# Output types
- RESPONSE
    - The text response generated by Qwen-VL based on the input media and prompt settings.
    - Comfy dtype: STRING
