# Documentation
- Class name: ImageRebatch
- Category: image/batch
- Output node: False
- Repo Ref: https://github.com/comfyanonymous/ComfyUI

The ImageRebatch node efficiently reorganizes image data into batches. It receives a set of images and a specified batch size, then processes these images to create smaller batches that can be used for parallel processing or batch‑based machine learning tasks. This node plays a key role in optimizing image data handling, enabling more efficient training and inference workflows.

# Input types
## Required
- images
    - The `images` parameter is the collection of image data that the node will process. It is the fundamental input, the raw material for batching. The node’s functionality directly depends on the quality and format of the input images, which will affect batching efficiency and results.
    - Comfy dtype: IMAGE
    - Python dtype: List[torch.Tensor]
- batch_size
    - The `batch_size` parameter determines how many images each output batch will contain. It is a critical parameter that sets the granularity of the batching process, directly influencing the node’s performance and subsequent processing or training steps. Choosing an appropriate batch size is important for balancing memory usage and computational efficiency.
    - Comfy dtype: INT
    - Python dtype: Tuple[int]

# Output types
- output_list
    - `output_list` is the list of image batches generated by the node. Each batch is a tensor of images prepared according to the specified batch size. This output is important because it represents the node’s primary deliverable, which can be further processed or fed into subsequent nodes or models in a machine‑learning pipeline.
    - Comfy dtype: IMAGE
    - Python dtype: List[torch.Tensor]
