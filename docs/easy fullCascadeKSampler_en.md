# Documentation
- Class name: samplerCascadeFull
- Category: EasyUse/Sampler
- Output node: True
- Repo Ref: https://github.com/yolain/ComfyUI-Easy-Use.git

The samplerCascadeFull node is designed to execute the sampling process within a generative model framework. It coordinates the encoding, sampling, and decoding stages to produce a sequence of images or latent representations. This node is essential for generating new image content based on provided seeds and models, offering users a seamless interface to leverage advanced sampling techniques.

# Input types
## Required
- pipe
    - The 'pipe' parameter is a critical input that carries the pipeline settings and data required for the sampling process. It includes loader settings, model references, and other necessary information to guide node operations.
    - Comfy dtype: PIPE_LINE
    - Python dtype: Dict[str, Any]
- encode_vae_name
    - The 'encode_vae_name' parameter specifies the name of the VAE model used to encode input images into the latent space. It is an optional parameter that allows customization of the encoding process.
    - Comfy dtype: STRING
    - Python dtype: Union[str, None]
- decode_vae_name
    - The 'decode_vae_name' parameter determines the VAE model used to decode latent representations back into images. It provides users with the flexibility to choose the decoding model according to their needs.
    - Comfy dtype: STRING
    - Python dtype: Union[str, None]
- steps
    - The 'steps' parameter defines the number of sampling steps to be executed during the sampling process. It directly affects the quality and detail of the generated images.
    - Comfy dtype: INT
    - Python dtype: int

# Output types
- pipe
    - The 'pipe' output contains the updated pipeline settings and results after the sampling process. It is a comprehensive output that encapsulates the state after node execution.
    - Comfy dtype: PIPE_LINE
    - Python dtype: Dict[str, Any]
- model_b
    - The 'model_b' output represents the model used during the decoding stage of the sampling process. It is important for users who need to reference the specific model used in their operations.
    - Comfy dtype: MODEL
    - Python dtype: torch.nn.Module
- latent_b
    - The 'latent_b' output provides the latent representations generated by the sampling process. These latent encodings are critical for users who wish to further manipulate or analyze the generated content.
    - Comfy dtype: LATENT
    - Python dtype: torch.Tensor
