# Documentation
- Class name: TransitionBetween2Image
- Category: AIGC
- Output node: False
- Repo Ref: https://github.com/esheep/esheep_custom_nodes.git

The TransitionBetween2Image node generates a smooth transition between two images. It uses Bezier curve interpolation to produce a series of frames that morph one image into another. This node is particularly useful for visual effects requiring gradual state changes, such as animation or video processing.

# Input types
## Required
- image_start
    - The starting image to transition from. It sets the initial visual state for the animation sequence.
    - Comfy dtype: IMAGE
    - Python dtype: PIL.Image.Image or numpy.ndarray
- image_end
    - The ending image to transition to. It represents the final visual state of the sequence.
    - Comfy dtype: IMAGE
    - Python dtype: PIL.Image.Image or numpy.ndarray
- total_frames
    - The total number of frames to generate for the transition. Increasing this number makes the transition smoother but requires more processing time.
    - Comfy dtype: INT
    - Python dtype: int
- begin_and_end_frames
    - The number of frames to duplicate at the beginning and end of the transition to provide a pause effect.
    - Comfy dtype: INT
    - Python dtype: int
- beiser_point_x
    - The x-coordinate of the control point on the Bezier curve that defines the transition shape.
    - Comfy dtype: FLOAT
    - Python dtype: float
- beiser_point_y
    - The y-coordinate of the control point on the Bezier curve that defines the transition shape.
    - Comfy dtype: FLOAT
    - Python dtype: float

# Output types
- IMAGE
    - The sequence of frames generated by the transition, which can be used for further processing or display.
    - Comfy dtype: IMAGE
    - Python dtype: torch.Tensor
