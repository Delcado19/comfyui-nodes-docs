# Documentation
- Class name: IPAdapterEncoder
- Category: ipadapter/embeds
- Output node: False
- Repo Ref: https://github.com/cubiq/ComfyUI_IPAdapter_plus.git

The IPAdapterEncoder node is designed to process and encode image data using a pre-trained CLIPVision model. It leverages the power of the CLIPVision architecture to generate embeddings that capture semantic information in images. The node is capable of processing both conditional and unconditional embeddings based on the presence of masks and weight parameters, allowing fine-tuning of the influence of image content on the generated embeddings.

# Input types
## Required
- ipadapter
    - The ipadapter parameter is critical to the node's operation, as it provides the model used for encoding image data. It is expected to be a dictionary containing the model information required for the encoding process. The presence of this parameter ensures the node can access the necessary model to generate embeddings.
    - Comfy dtype: IPADAPTER
    - Python dtype: Dict[str, Any]
- image
    - The image parameter is required, as it represents the input data the node will process. It is expected to be an image tensor that the node will encode into embeddings. The quality and format of the image data directly impact the node's ability to generate meaningful embeddings.
    - Comfy dtype: IMAGE
    - Python dtype: torch.Tensor
- weight
    - The weight parameter allows adjusting the influence of image content on the generated embeddings. It is a float that, when not equal to 1, can scale the embeddings to emphasize or de-emphasize certain aspects of the image data.
    - Comfy dtype: FLOAT
    - Python dtype: float
## Optional
- mask
    - The optional mask parameter can be used to apply a spatial mask to the image data before encoding. This can be useful for focusing the node's attention on specific regions of the image or excluding irrelevant parts of the image from the encoding process.
    - Comfy dtype: MASK
    - Python dtype: torch.Tensor
- clip_vision
    - The clip_vision parameter is an optional model that can be provided to the node for encoding image data. If not provided, the node will use the model specified in the ipadapter parameter. This allows using different CLIPVision models for different encoding tasks.
    - Comfy dtype: CLIP_VISION
    - Python dtype: Any

# Output types
- pos_embed
    - The pos_embed output represents the conditional embeddings generated by the node. These embeddings are influenced by the image data and any masks applied, capturing semantic information in the image.
    - Comfy dtype: EMBEDS
    - Python dtype: torch.Tensor
- neg_embed
    - The neg_embed output provides unconditional embeddings, which are generated without considering the image content. These embeddings can serve as a baseline or reference point for comparison with the conditional embeddings.
    - Comfy dtype: EMBEDS
    - Python dtype: torch.Tensor
