# Documentation
- Class name: BatchStringSchedule
- Category: FizzNodes 📅🅕🅝/BatchScheduleNodes
- Output node: False
- Repo Ref: https://github.com/FizzleDorf/ComfyUI_FizzNodes

The `animate` method of the BatchStringSchedule node is designed to process structured input text and generate text animation. It uses a series of weighted sub‑prompts to create dynamic prompt sequences that can be used for animation or generating text‑based content. This method is essential for applications that need to transform or manipulate text across a series of frames or steps.

# Input types
## Required
- text
- The `text` parameter is the node's primary input, containing structured text that defines the animation sequence. It is critical because it directly influences the output prompts and the entire animation process.
    - Comfy dtype: STRING
    - Python dtype: str
- max_frames
- The `max_frames` parameter specifies the maximum number of frames in the animation sequence. It plays an important role in determining the animation's duration and granularity.
    - Comfy dtype: INT
    - Python dtype: int
## Optional
- print_output
- When the `print_output` parameter is set to True, intermediate results of the animation process are printed to the console. This is useful for debugging and understanding the animation generation workflow.
    - Comfy dtype: BOOLEAN
    - Python dtype: bool
- pre_text
- The `pre_text` parameter adds additional context at the beginning of each prompt in the animation sequence. This can be important for establishing the initial state or conditions of the animation.
    - Comfy dtype: STRING
    - Python dtype: str
- app_text
- The `app_text` parameter is appended to the end of each prompt in the animation sequence, potentially adding more context or instructions to the generated prompts.
    - Comfy dtype: STRING
    - Python dtype: str
- pw_a
- The `pw_a` parameter is one of the weight parameters that affect interpolation and blending of prompts in the animation sequence. It helps smoothness and transition between frames.
    - Comfy dtype: FLOAT
    - Python dtype: float
- pw_b
- The `pw_b` parameter is another weight parameter that acts during interpolation, influencing the balance and distribution of prompt elements in animation frames.
    - Comfy dtype: FLOAT
    - Python dtype: float
- pw_c
- The `pw_c` parameter is a weight parameter that fine‑tunes interpolation, allowing more precise control over the output prompts.
    - Comfy dtype: FLOAT
    - Python dtype: float
- pw_d
- The `pw_d` parameter is the final weight affecting interpolation, providing additional control over how prompts evolve during the animation.
    - Comfy dtype: FLOAT
    - Python dtype: float

# Output types
- POS
- The `POS` output contains positive prompts generated by the animation process, representing constructive or desired aspects of the text animation.
    - Comfy dtype: STRING
    - Python dtype: str
- NEG
- The `NEG` output contains negative prompts generated by the animation process, representing aspects to be minimized or avoided in the text animation.
    - Comfy dtype: STRING
    - Python dtype: str
