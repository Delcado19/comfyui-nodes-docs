# Documentation
- Class name: IterativeMixingSchedulerAdvanced
- Category: sampling/custom_sampling/schedulers
- Output node: False
- Repo Ref: https://github.com/ttulttul/ComfyUI-Iterative-Mixer

The IterativeMixingSchedulerAdvanced node is designed to generate a series of noise levels, i.e., sigmas, for the iterative sampling process. It intelligently adjusts sampling steps based on the denoising parameter, ensuring the output is fine-tuned to the desired noise reduction level. This node plays a key role in controlling the sampling schedule of advanced iterative sampling techniques, contributing to the overall quality and efficiency of the sampling process.

# Input types
## Required
- model
    - The model parameter is critical to the node's operation, as it defines the generative model to which the scheduler will be applied. It directly affects the quality and type of sigmas generated, which is essential for the sampling process.
    - Comfy dtype: MODEL
    - Python dtype: comfy.model_management.Model
- scheduler
    - The scheduler parameter determines the scheduling strategy to be used during the sampling process. It is a key factor in computing sigmas, and thus influences the overall performance of iterative sampling.
    - Comfy dtype: SCHEDULER_NAMES
    - Python dtype: str
- steps
    - The steps parameter specifies the number of iterations or steps to be performed during the sampling process. It is an important factor in controlling the duration and granularity of the sampling schedule.
    - Comfy dtype: INT
    - Python dtype: int
- start_at_step
    - The start_at_step parameter allows customization of the starting point of the sampling schedule. It can be used to skip initial steps or begin sampling at a specific point in the process.
    - Comfy dtype: INT
    - Python dtype: int
- end_at_step
    - The end_at_step parameter defines the endpoint of the sampling schedule. It can be used to limit the number of steps executed, stopping the sampling process at a predetermined point.
    - Comfy dtype: INT
    - Python dtype: int
- denoise
    - The denoise parameter adjusts the level of noise reduction during the sampling process. It is a key factor in determining the quality of the final output, achieved by controlling the strength of the denoising steps.
    - Comfy dtype: FLOAT
    - Python dtype: float

# Output types
- sigmas
    - The sigmas output represents the sequence of noise levels generated by the node. These values guide the iterative sampling process, steering the model toward producing higher-quality output.
    - Comfy dtype: SIGMAS
    - Python dtype: torch.Tensor
