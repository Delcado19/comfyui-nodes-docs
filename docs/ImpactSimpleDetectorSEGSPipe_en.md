# Documentation
- Class name: SimpleDetectorForEachPipe
- Category: ImpactPack/Detector
- Output node: False
- Repo Ref: https://github.com/ltdrdata/ComfyUI-Impact-Pack.git

The SimpleDetectorForEachPipe node processes images through a predefined detection pipeline. It considers various parameters that affect the detection process, such as bounding‑box threshold and dilation, to improve detection accuracy. The node’s function focuses on identifying and locating objects in images, aiding the broader goal of image analysis and object recognition.

# Input types
## Required
- detailer_pipe
- The `detailer_pipe` parameter is essential for the node’s operation because it defines the series of operations or pipeline applied to the input image for detection. It is a key component that determines the node’s behavior and the quality of detection results.
    - Comfy dtype: DETAILER_PIPE
    - Python dtype: Tuple[Any, ...]
- image
- The `image` parameter is the node’s main input, representing the data to be processed for object detection. Its importance lies in being the subject of the detection operation; the node’s performance and results directly depend on the quality and characteristics of the input image.
    - Comfy dtype: IMAGE
    - Python dtype: torch.Tensor
- bbox_threshold
- The `bbox_threshold` parameter determines the confidence level at which a bounding box is considered a valid detection. It plays a crucial role in filtering out false positives and ensuring that only high‑confidence detections are included in the output.
    - Comfy dtype: FLOAT
    - Python dtype: float
- bbox_dilation
- The `bbox_dilation` parameter controls the amount of dilation applied to bounding boxes, helping adjust the size of detected objects. This parameter is important for fine‑tuning detection accuracy to better meet the specific requirements of the application.
    - Comfy dtype: INT
    - Python dtype: int
- crop_factor
- The `crop_factor` parameter determines the portion of the image to be considered for detection. It is important because it can focus the detection process on the most relevant region of the image, potentially improving the node’s efficiency and accuracy.
    - Comfy dtype: FLOAT
    - Python dtype: float
- drop_size
- The `drop_size` parameter specifies the size of the grid cells into which the image is divided for detection purposes. It affects the granularity of detection and is crucial for determining the level of detail in the detection results.
    - Comfy dtype: INT
    - Python dtype: int
- sub_threshold
- The `sub_threshold` parameter sets the threshold for secondary detections, which can refine the detection process by focusing on smaller or less prominent objects in the image.
    - Comfy dtype: FLOAT
    - Python dtype: float
- sub_dilation
- The `sub_dilation` parameter allows the size of secondary detections to be adjusted by applying dilation. It is important for controlling the scale of detected objects and can be used to optimize detection for specific scenarios.
    - Comfy dtype: INT
    - Python dtype: int
- sub_bbox_expansion
- The `sub_bbox_expansion` parameter controls the expansion of bounding boxes for secondary detections. It is important because it can capture a broader context around smaller objects, potentially improving detection accuracy.
    - Comfy dtype: INT
    - Python dtype: int
- sam_mask_hint_threshold
- The `sam_mask_hint_threshold` parameter defines the threshold for mask hints generated by the SAM model. It is important for controlling the level of detail in segmentation masks, which can affect the final detection results.
    - Comfy dtype: FLOAT
    - Python dtype: float
## Optional
- post_dilation
- The `post_dilation` parameter provides an additional layer of control over the final bounding boxes by allowing dilation after the main detection process. It can be used to fine‑tune detection results for specific use cases.
    - Comfy dtype: INT
    - Python dtype: Optional[int]

# Output types

- The `segs` output parameter represents the segmentation regions detected in the input image. It is important because it contains the primary results of the detection process, offering valuable insights into the composition and structure of the image.
