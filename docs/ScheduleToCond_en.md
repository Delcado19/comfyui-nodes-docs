# Documentation
- Class name: ScheduleToCond
- Category: promptcontrol
- Output node: False
- Repo Ref: https://github.com/asagi4/comfyui-prompt-control.git

The 'apply' method of the ScheduleToCond node is designed to process and apply a given prompt schedule to a CLIP model. It integrates the prompt schedule with the CLIP model to generate a set of conditioning inputs, which can be used for further processing or as part of the model input. This method plays a key role in controlling the influence of text prompts on the CLIP model's output.

# Input types
## Required
- clip
    - The 'clip' parameter is critical because it represents the CLIP model to which the prompt schedule will be applied. It is the core component that determines the subsequent conditioning processing and the output generated by the node.
    - Comfy dtype: CLIP
    - Python dtype: torch.nn.Module
- prompt_schedule
    - The 'prompt_schedule' parameter defines the schedule for applying prompts to the CLIP model. It is essential for determining how text prompts will modulate the model's behavior during processing.
    - Comfy dtype: PROMPT_SCHEDULE
    - Python dtype: Dict[str, Any]

# Output types
- CONDITIONING
    - The 'CONDITIONING' output represents the set of conditions derived from the prompt schedule and applied to the CLIP model. It is important because it forms the foundation for further operations or serves as input for downstream tasks.
    - Comfy dtype: CONDITIONING
    - Python dtype: List[Tuple[str, Dict[str, Any]]]
