# Documentation
- Class name: samplerSimpleTiled
- Category: EasyUse/Sampler
- Output node: True
- Repo Ref: https://github.com/yolain/ComfyUI-Easy-Use.git

This node simplifies the sampling process by managing image tiling and processing output according to user preferences. It abstracts the complexity of sampling, allowing users to focus on creative input without worrying about the technical details of the sampling process.

# Input types
## Required
- pipe
    - The pipeline parameter is required because it carries the state of the entire pipeline, including model and data information, guiding node execution.
    - Comfy dtype: PIPE_LINE
    - Python dtype: Dict
- tile_size
    - The tile size parameter determines the dimensions of the tiled output, which is critical for controlling the resolution and layout of the final image.
    - Comfy dtype: INT
    - Python dtype: int
- image_output
    - This parameter controls how image output is handled, whether for preview, saving, or other purposes, affecting node availability and workflow.
    - Comfy dtype: COMBO
    - Python dtype: Union[str, None]
- link_id
    - The link ID parameter is important for establishing connections between different parts of the system, ensuring correct data flow and communication.
    - Comfy dtype: INT
    - Python dtype: int
- save_prefix
    - This parameter specifies the prefix for saved files, which is essential for organizing and identifying outputs in the file system.
    - Comfy dtype: STRING
    - Python dtype: str
## Optional
- model
    - When a model parameter is provided, it allows the node to use a specific model during the sampling process, affecting the quality and characteristics of the generated image.
    - Comfy dtype: MODEL
    - Python dtype: Any
- prompt
    - The prompt parameter is used to guide the generation process, and its content significantly influences the creative direction and results of sampling.
    - Comfy dtype: PROMPT
    - Python dtype: Any
- extra_pnginfo
    - This parameter contains additional information related to PNG images, which can enhance the processing and handling of such images within the node.
    - Comfy dtype: EXTRA_PNGINFO
    - Python dtype: Any
- my_unique_id
    - The unique ID parameter is used to track and manage individual instances of node operations, ensuring personalized and targeted execution.
    - Comfy dtype: UNIQUE_ID
    - Python dtype: Any

# Output types
- pipe
    - The pipeline output is a comprehensive structure that includes the updated pipeline state, reflecting changes made during the sampling process.
    - Comfy dtype: PIPE_LINE
    - Python dtype: Dict
- image
    - The image output represents the visual result of sampling and is the primary creative content generated by the node.
    - Comfy dtype: IMAGE
    - Python dtype: torch.Tensor
