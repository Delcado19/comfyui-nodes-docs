# Documentation
- Class name: AILab_QwenVL_GGUF_Advanced
- Display name: QwenVL Advanced (GGUF)
- Category: 🧪AILab/QwenVL
- Output node: False
- Repo Ref: https://github.com/1038lab/ComfyUI-QwenVL

Runs QwenVL GGUF vision-language model inference on images or video frames, returning a text response based on the selected or custom prompt.

# Input types
## Required
- model_name
    - Selects which QwenVL GGUF model file to load for inference.
    - Comfy dtype: COMBO
    - Default: `"Qwen3VL-4B-Instruct-F16.gguf"`
    - Options: populated at runtime from installed files
- device
    - Chooses the compute device used for model execution (e.g., auto, CPU, or specific GPU).
    - Comfy dtype: COMBO
    - Default: `"auto"`
    - Options: `auto`, `cpu`, `mps`, `cuda:0`
- preset_prompt
    - Selects a predefined prompt template to guide the model's response style.
    - Comfy dtype: COMBO
    - Default: `"🖼️ Detailed Description"`
    - Options: `🖼️ Tags`, `🖼️ Simple Description`, `🖼️ Detailed Description`, `🖼️ Ultra Detailed Description`, `🎬 Cinematic Description`, `🖼️ Detailed Analysis`, `📹 Video Summary`, `📖 Short Story`, `🪄 Prompt Refine & Expand`
- custom_prompt
    - Overrides the preset with a fully custom text prompt for the model.
    - Comfy dtype: STRING
    - Default: `""`
- max_tokens
    - Sets the maximum number of tokens the model can generate in its response.
    - Comfy dtype: INT
    - Default: `512`
- temperature
    - Controls randomness in token generation; lower values produce more deterministic output.
    - Comfy dtype: FLOAT
    - Default: `0.6`
- top_p
    - Sets the nucleus sampling threshold for token selection diversity.
    - Comfy dtype: FLOAT
    - Default: `0.9`
- repetition_penalty
    - Penalizes repeated tokens to reduce looping in the output.
    - Comfy dtype: FLOAT
    - Default: `1.2`
- frame_count
    - Number of video frames to sample when processing video input.
    - Comfy dtype: INT
    - Default: `16`
- ctx
    - Sets the context window size (in tokens) for the model.
    - Comfy dtype: INT
    - Default: `8192`
- n_batch
    - Number of tokens processed per batch during prompt evaluation.
    - Comfy dtype: INT
    - Default: `512`
- gpu_layers
    - Number of model layers offloaded to GPU; -1 loads the full model to GPU.
    - Comfy dtype: INT
    - Default: `-1`
- image_max_tokens
    - Maximum tokens allocated for image encoding.
    - Comfy dtype: INT
    - Default: `4096`
- top_k
    - Limits token selection to the top K most likely candidates; 0 disables top-k sampling.
    - Comfy dtype: INT
    - Default: `0`
- pool_size
    - Sets the memory pool size (in bytes) for the inference backend.
    - Comfy dtype: INT
    - Default: `4194304`
- keep_model_loaded
    - Keeps the model in memory between executions to avoid reload overhead.
    - Comfy dtype: BOOLEAN
    - Default: `true`
- seed
    - Sets the random seed for reproducible generation.
    - Comfy dtype: INT
    - Default: `1`
## Optional
- image
    - A single image to analyze with the model.
    - Comfy dtype: IMAGE
- video
    - A video (as an image sequence) to analyze with the model.
    - Comfy dtype: IMAGE

# Output types
- RESPONSE
    - The text response generated by the QwenVL model based on the input and prompt.
    - Comfy dtype: STRING
