
# Documentation
- Class name: Eff. Loader SDXL
- Category: Efficiency Nodes/Loaders
- Output node: False

The Eff. Loader SDXL node is designed to efficiently load and prepare models and data for the SDXL (Super Diffusion XL) pipeline. It handles initialization and configuration of the base model and refiner model, processing various parameters such as clip skip and aesthetic scores to optimize the loading process, thereby improving performance and efficiency for generation tasks.

# Input types
## Required
- base_ckpt_name
    - Specifies the checkpoint name for the base model, essential for initializing the model with pretrained weights.
    - Comfy dtype: COMBO[STRING]
    - Python dtype: str
- base_clip_skip
    - Determines the number of clip layers to skip in the base model, affecting model performance and output quality.
    - Comfy dtype: INT
    - Python dtype: int
- refiner_ckpt_name
    - Defines the checkpoint name for the refiner model, enabling enhancement of generated outputs through refinement.
    - Comfy dtype: COMBO[STRING]
    - Python dtype: str
- refiner_clip_skip
    - Indicates the number of clip layers to skip in the refiner model, used to optimize the refinement process.
    - Comfy dtype: INT
    - Python dtype: int
- positive_ascore
    - Aesthetic score for positive feedback, used to guide the model toward generating more appealing outputs.
    - Comfy dtype: FLOAT
    - Python dtype: float
- negative_ascore
    - Aesthetic score for negative feedback, helping the model avoid less desirable outputs.
    - Comfy dtype: FLOAT
    - Python dtype: float
- vae_name
    - The name of the VAE used in the process, critical for the model's data encoding and decoding.
    - Comfy dtype: COMBO[STRING]
    - Python dtype: str
- positive
    - Positive input data or conditioning, used to steer the model toward specific generation goals.
    - Comfy dtype: STRING
    - Python dtype: str
- negative
    - Negative input data or conditioning, used to prevent certain patterns or themes from appearing in generated outputs.
    - Comfy dtype: STRING
    - Python dtype: str
- token_normalization
    - Normalization of conditioning tokens, affecting how the model interprets and processes input data.
    - Comfy dtype: COMBO[STRING]
    - Python dtype: bool
- weight_interpretation
    - Adjusts the weight with which the model interprets different inputs, influencing the balance between positive and negative feedback.
    - Comfy dtype: COMBO[STRING]
    - Python dtype: bool
- empty_latent_width
    - Specifies the width of the blank latent space, configuring the dimensions for model initialization.
    - Comfy dtype: INT
    - Python dtype: int
- empty_latent_height
    - Defines the height of the blank latent space, setting the spatial dimensions for the generation process.
    - Comfy dtype: INT
    - Python dtype: int
- batch_size
    - The number of samples processed in a single batch, affecting the efficiency and speed of loading operations.
    - Comfy dtype: INT
    - Python dtype: int
## Optional
- lora_stack
    - Optional LoRA stack configuration, enhancing model performance through LoRA (Low-Rank Adaptation) techniques.
    - Comfy dtype: LORA_STACK
    - Python dtype: str
- cnet_stack
    - Optional ControlNet stack configuration, further optimizing model outputs through controlled network adjustments.
    - Comfy dtype: CONTROL_NET_STACK
    - Python dtype: str

# Output types
- SDXL_TUPLE
    - Comfy dtype: SDXL_TUPLE
    - A tuple containing the base model and refiner model, clips, and encoded positive and negative conditioning, ready for the SDXL pipeline.
    - Python dtype: tuple
- LATENT
    - Comfy dtype: LATENT
    - Latent space samples generated by the VAE, critical to the generation process.
    - Python dtype: dict
- VAE
    - Comfy dtype: VAE
    - The VAE model used in the process, essential for data encoding and decoding.
    - Python dtype: object
- DEPENDENCIES
    - Comfy dtype: DEPENDENCIES
    - A collection of dependencies required for the operation, including model names, clip information, and configuration parameters.
    - Python dtype: tuple
