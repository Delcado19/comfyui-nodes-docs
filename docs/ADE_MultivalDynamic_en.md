# Documentation
- Class name: MultivalDynamicNode
- Category: Animate Diff 🎭🅐🅓/multival
- Output node: False
- Repo Ref: https://github.com/Kosinkadink/ComfyUI-AnimateDiff-Evolved.git

The MultivalDynamicNode class is designed to dynamically generate and manipulate multivalue data within an animation framework. It is a key component for creating complex animations, allowing users to specify a range of values that can be applied to different aspects of the animation, such as character movement or scene transitions. The node's functionality revolves around the 'create_multival' method, which intelligently handles both single and iterable input values and ensures the output format is suitable for downstream processing.

# Input types
## Required
- float_val
    - The parameter 'float_val' is crucial for defining the base value or sequence of values that the node will use to create multivalue data. It can accept both a single floating-point number and an iterable of floating-point numbers, providing a high degree of flexibility in different aspects of the animation scene. This parameter has a significant impact on the node's execution, as it directly affects the range and nature of the output values.
    - Comfy dtype: FLOAT
    - Python dtype: Union[float, List[float]]
## Optional
- mask_optional
    - The parameter 'mask_optional' is used to apply a mask to the multivalue data generated by the node. It is an optional tensor that, when provided, allows the selective application of 'float_val' on different elements of the animation. This parameter enhances the node's functionality by enabling conditional or partial animation effects, adding depth to the final animation.
    - Comfy dtype: MASK
    - Python dtype: Optional[torch.Tensor]

# Output types
- multival
    - The output 'multival' represents the processed multivalue data, ready for use in the animation pipeline. It is a tensor that contains the dynamic range of values specified by the 'float_val' parameter, possibly modified by the 'mask_optional' parameter. This output is important because it forms the basis for further animation development and can be directly integrated into the rendering process.
    - Comfy dtype: MULTIVAL
    - Python dtype: torch.Tensor
