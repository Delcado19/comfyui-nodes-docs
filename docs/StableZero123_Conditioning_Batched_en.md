# Documentation
- Class name: StableZero123_Conditioning_Batched
- Category: conditioning/3d_models
- Output node: False
- Repo Ref: https://github.com/comfyanonymous/ComfyUI

This node aims to batch process and generate conditioning data for 3D models, integrating visual and spatial information to enhance model expressiveness.

# Input types
## Required
- clip_vision
    - The clip_vision parameter is essential for encoding visual information from images, which is critical for the node to produce accurate and meaningful conditioning data.
    - Comfy dtype: CLIP_VISION
    - Python dtype: comfy.model_patcher.ModelPatcher
- init_image
    - The init_image parameter serves as the foundation for node operations, providing the initial visual context that is upscaled and encoded for further processing.
    - Comfy dtype: IMAGE
    - Python dtype: PIL.Image.Image
- vae
    - The vae parameter is crucial for encoding visual data into the latent space, enabling the node to efficiently generate conditioning data for manipulating 3D models.
    - Comfy dtype: VAE
    - Python dtype: torch.nn.Module
- width
    - The width parameter determines the horizontal resolution of the upscaled image, affecting the level of detail and complexity of the conditioning data generated by the node.
    - Comfy dtype: INT
    - Python dtype: int
- height
    - The height parameter sets the vertical resolution of the upscaled image, directly impacting the quality and richness of the 3D model conditioning data.
    - Comfy dtype: INT
    - Python dtype: int
- batch_size
    - The batch_size parameter determines the number of operations the node can execute simultaneously, improving the efficiency and throughput of the 3D model conditioning process.
    - Comfy dtype: INT
    - Python dtype: int
- elevation
    - The elevation parameter specifies the vertical angle of the camera, which is important for creating realistic and contextually accurate 3D model conditioning data.
    - Comfy dtype: FLOAT
    - Python dtype: float
- azimuth
    - The azimuth parameter defines the horizontal angle of the camera, influencing the perspective and orientation of the 3D model conditioning data.
    - Comfy dtype: FLOAT
    - Python dtype: float
- elevation_batch_increment
    - The elevation_batch_increment parameter allows batch adjustments to the camera's vertical angle, ensuring diversity in the conditioning data for 3D models.
    - Comfy dtype: FLOAT
    - Python dtype: float
- azimuth_batch_increment
    - The azimuth_batch_increment parameter facilitates batch adjustments to the camera's horizontal angle, contributing to the diversity of 3D model conditioning data.
    - Comfy dtype: FLOAT
    - Python dtype: float

# Output types
- positive
    - The positive output provides a set of conditioning data representing the desired state of the 3D model, serving as guidance for model generation and manipulation.
    - Comfy dtype: CONDITIONING
    - Python dtype: List[Tuple[torch.Tensor, Dict[str, torch.Tensor]]]
- negative
    - The negative output provides a complementary set of conditioning data defining the undesired state of the 3D model, aiding in refining and constraining the generation process.
    - Comfy dtype: CONDITIONING
    - Python dtype: List[Tuple[torch.Tensor, Dict[str, torch.Tensor]]]
- latent
    - The latent output contains encoded spatial and visual information, forming the basis for the internal representation and manipulation of the 3D model within the system.
    - Comfy dtype: LATENT
    - Python dtype: torch.Tensor
