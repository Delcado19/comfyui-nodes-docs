# Documentation
- Class name: WildcardOobaPrompt
- Category: Mikey/AI
- Output node: True
- Repo Ref: https://github.com/bash-j/mikey_nodes

The 'process' method of the WildcardOobaPrompt node is designed to dynamically generate and process prompts for AI applications. It replaces wildcards in the input prompt with random selections and integrates with external APIs to generate detailed responses. This node plays a key role in creating diverse and context-rich prompts for AI models, enhancing their ability to produce nuanced outputs.

# Input types
## Required
- input_prompt
    - The ‘input_prompt’ parameter is a string containing the initial text of the prompt. It is crucial because it forms the basis for wildcard replacement and subsequent AI processing. This parameter directly affects the diversity and creativity of the final output generated by the node.
    - Comfy dtype: STRING
    - Python dtype: str
- seed
    - The ‘seed’ parameter is an integer used to initialize the random number generator in the wildcard replacement process. It ensures that results are reproducible, which is crucial for debugging and consistency testing of the node's functionality.
    - Comfy dtype: INT
    - Python dtype: int
## Optional
- unique_id
    - The ‘unique_id’ parameter is a unique identifier for referencing specific entries in the prompt. It is especially important when managing multiple concurrent prompts, as it allows the node to precisely locate and update the required prompt.
    - Comfy dtype: UNIQUE_ID
    - Python dtype: str

# Output types
- text
    - The 'text' output parameter represents the final processed prompt after all wildcards have been replaced and the AI response has been integrated. It is a critical output because it is directly input into the AI model, affecting the quality and characteristics of the generated content.
    - Comfy dtype: STRING
    - Python dtype: str
