# Documentation
- Class name: BasicPipeToDetailerPipeSDXL
- Category: ImpactPack/Pipe
- Output node: False
- Repo Ref: https://github.com/ltdrdata/ComfyUI-Impact-Pack.git

The BasicPipeToDetailerPipeSDXL node converts a basic pipeline structure into a more detailed form suitable for advanced processing stages. It enhances the pipeline's capabilities for tasks like segmentation and refinement by integrating components such as bbox_detector, LoRA, and Wildcard selection.

# Input types
## Required
- base_basic_pipe
    - The base_basic_pipe parameter is required, as it provides the foundational pipeline structure the node needs to operate. This is a critical input that directly affects the node's ability to process and transform data.
    - Comfy dtype: BASIC_PIPE
    - Python dtype: Tuple[Any, ...]
- refiner_basic_pipe
    - The refiner_basic_pipe parameter refines the base pipeline, providing an additional layer of processing capability. It is a key component for improving the detail and accuracy of the pipeline's output.
    - Comfy dtype: BASIC_PIPE
    - Python dtype: Tuple[Any, ...]
- bbox_detector
    - The bbox_detector is a critical component for identifying and locating regions of interest in the data. Its role is essential for enabling the node to perform spatial analysis and object detection tasks.
    - Comfy dtype: BBOX_DETECTOR
    - Python dtype: Any
- wildcard
    - The wildcard parameter allows dynamic input patterns, enhancing the node's flexibility. It is significant for adapting to various data formats and processing requirements.
    - Comfy dtype: STRING
    - Python dtype: str
