# Documentation
- Class name: MiDaS_Depth_Approx
- Category: WAS Suite/Image/AI
- Output node: False
- Repo Ref: https://github.com/WASasquatch/was-node-suite-comfyui

The method `midas_approx` is responsible for estimating the depth of an image using the MiDaS model. It processes the input image to predict a depth map, which is useful in various computer vision tasks that require understanding the spatial structure of a scene. The method is designed to be flexible, allowing users to specify whether to use CPU or GPU, select the type of MiDaS model, and choose whether to invert the depth for specific applications.

# Input types
## Required
- image
    - The input image parameter is critical to the operation of the node, as it is the primary data the node processes to generate the depth approximation. The quality and resolution of the input image directly affect the accuracy and detail of the generated depth map.
    - Comfy dtype: IMAGE
    - Python dtype: torch.Tensor
## Optional
- use_cpu
    - The 'use_cpu' parameter determines whether the node should use the CPU for processing. This is important for systems without a GPU or when users want to conserve GPU resources for other tasks.
    - Comfy dtype: COMBO['false', 'true']
    - Python dtype: str
- midas_type
    - The 'midas_type' parameter allows users to select different MiDaS model architectures. Depending on the specific requirements of the application, the choice can affect the balance between accuracy and processing speed.
    - Comfy dtype: COMBO['DPT_Large', 'DPT_Hybrid']
    - Python dtype: str
- invert_depth
    - The 'invert_depth' parameter provides an option to invert the depth values predicted by the model. This can be useful in scenarios where higher depth values are needed to represent closer objects, which is the opposite of the standard depth map convention.
    - Comfy dtype: COMBO['false', 'true']
    - Python dtype: str
- midas_model
    - The optional 'midas_model' parameter allows the use of a pre-loaded MiDaS model, eliminating the need to load the model within the node itself. This can simplify the processing workflow, especially when reusing the same model.
    - Comfy dtype: MIDAS_MODEL
    - Python dtype: Tuple[torch.nn.Module, Callable]

# Output types
- images
    - The output 'images' parameter represents the depth approximation images generated by the node. These images are critical for downstream tasks that rely on understanding the spatial dimensions of the scene in the input image.
    - Comfy dtype: IMAGE
    - Python dtype: torch.Tensor
