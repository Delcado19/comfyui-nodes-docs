# Documentation
- Class name: ImageUpscaleWithModel
- Category: image/upscaling
- Output node: False
- Repo Ref: https://github.com/comfyanonymous/ComfyUI

The ImageUpscaleWithModel node enhances the resolution of input images using a specified upscale model. It manages system memory intelligently to prevent out‑of‑memory errors, ensuring a smooth upscaling process. The node focuses on improving image quality without sacrificing performance.

# Input types
## Required
- upscale_model
- The upscale_model parameter is essential to the node because it defines the model used for upscaling. It directly affects the upsampling process and the quality of the resulting image.
    - Comfy dtype: torch.nn.Module
    - Python dtype: torch.nn.Module
- image
- The image parameter is required because it represents the input image data the node will process. Its structure and content are critical to the success of the upsampling operation.
    - Comfy dtype: torch.Tensor
    - Python dtype: torch.Tensor

# Output types
- IMAGE
- The output parameter IMAGE represents the upscaled image generated by the node. It marks the successful completion of the upsampling process and is the primary outcome of the node's functionality.
    - Comfy dtype: torch.Tensor
    - Python dtype: torch.Tensor
