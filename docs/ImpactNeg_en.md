# Documentation
- Class name: ImpactNeg
- Category: ImpactPack/Logic
- Output node: False
- Repo Ref: https://github.com/ltdrdata/ComfyUI-Impact-Pack.git

The 'doit' method of the ImpactNeg node is designed to invert the logical state of a boolean input. It is a fundamental building block in logical operations, providing a direct yet crucial function within the ImpactPack suite. The role of this node is to ensure the output is the opposite of the input, contributing to a broader logical framework by providing a way to negate conditions.

# Input types
## Required
- value
    - The 'value' parameter is the core component of the ImpactNeg node's functionality. It represents the boolean condition that the node will negate. The importance of this parameter lies in its direct impact on the node's output, as the result depends entirely on the logical state of the input value.
    - Comfy dtype: BOOLEAN
    - Python dtype: bool

# Output types
- result
    - The 'result' parameter is the outcome of the ImpactNeg node's operation. It is the logical NOT of the input 'value' boolean. This output is important as it directly reflects the negation purpose of the node, providing a clear, concise boolean result for further use in logical expressions or conditions.
    - Comfy dtype: BOOLEAN
    - Python dtype: bool
