# Documentation
- Class name: VAEDecodeTiled
- Category: _for_testing
- Output node: False
- Repo Ref: https://github.com/comfyanonymous/ComfyUI

The VAEDecodeTiled node decodes latent representations into images using a Variational Autoencoder (VAE) model. It performs this decoding in a tiled manner, which is beneficial for processing large images or when computational resources are limited. The primary goal of the node is to provide a memory-efficient and scalable method for generating images from latent codes.

# Input types
## Required
- samples
    - The 'samples' parameter is critical because it contains the latent representations that the node will decode into images. This is a required input that directly affects the node's output, determining the characteristics of the generated images.
    - Comfy dtype: LATENT
    - Python dtype: torch.Tensor
- vae
    - The 'vae' parameter specifies the Variational Autoencoder model that the node will use to decode the latent representations. This parameter is necessary because it determines the architecture and parameters of the model, which in turn affects the quality and style of the decoded images.
    - Comfy dtype: VAE
    - Python dtype: AutoencoderKL
## Optional
- tile_size
    - The 'tile_size' parameter determines the size of the tiles into which the image will be divided during decoding. It is an optional input that can be adjusted based on memory usage and processing time, allowing for more flexible handling of images of different sizes.
    - Comfy dtype: INT
    - Python dtype: int

# Output types
- output
    - The 'output' parameter represents the decoded image generated by the node. It is the primary result of the node's execution, and is significant because it reflects the outcome of the decoding process.
    - Comfy dtype: IMAGE
    - Python dtype: torch.Tensor
