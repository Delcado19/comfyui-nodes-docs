# Documentation
- Class name: StyleAlignedReferenceSampler
- Category: style_aligned
- Output node: False
- Repo Ref: https://github.com/brianfitzgerald/style_aligned_comfy

The StyleAlignedReferenceSampler class is designed to facilitate style-aligned latent space sampling, ensuring that generated content follows specific style references. It integrates attention mechanisms and normalization techniques to refine outputs, providing a nuanced approach to style alignment in generative models.

# Input types
## Required
- model
    - The model parameter is critical as it defines the architecture and parameters of the generative model used for style-aligned sampling. It is the foundation upon which the style alignment and sampling process is built.
    - Comfy dtype: MODEL
    - Python dtype: ModelPatcher
- share_norm
    - The share_norm parameter determines how normalization layers are shared across different layers or groups in the model, which significantly impacts the style alignment process by controlling information flow.
    - Comfy dtype: SHARE_NORM_OPTIONS
    - Python dtype: str
- share_attn
    - The share_attn parameter determines the sharing strategy for attention mechanisms in the model, which is crucial for aligning generated content with the desired style by influencing attention weights.
    - Comfy dtype: SHARE_ATTN_OPTIONS
    - Python dtype: str
- scale
    - The scale parameter adjusts the influence of the style reference on generated samples, allowing fine-tuning of style alignment to achieve desired aesthetic outcomes.
    - Comfy dtype: FLOAT
    - Python dtype: float
- batch_size
    - The batch_size determines the number of samples processed simultaneously, which affects the efficiency and speed of the style-aligned sampling process.
    - Comfy dtype: INT
    - Python dtype: int
- noise_seed
    - The noise_seed is used to generate noise vectors during the sampling process, ensuring that style-aligned samples can be consistently reproduced.
    - Comfy dtype: INT
    - Python dtype: int
- cfg
    - The cfg parameter influences the configuration of the sampling process, particularly how the model responds to conditional inputs, which is essential for achieving accurate style alignment.
    - Comfy dtype: FLOAT
    - Python dtype: float
- positive
    - The positive condition input is crucial for guiding the model toward desired style features, serving as a reference for the style alignment process.
    - Comfy dtype: CONDITIONING
    - Python dtype: T
- negative
    - The negative condition input helps refine the model's output by specifying undesirable features to avoid, improving the precision of style alignment.
    - Comfy dtype: CONDITIONING
    - Python dtype: T
- ref_positive
    - The ref_positive condition input provides an additional layer of style reference, further ensuring that generated samples closely match the target style.
    - Comfy dtype: CONDITIONING
    - Python dtype: T
- sampler
    - The sampler parameter defines the sampling strategy, which is critical for the diversity and quality of style-aligned outputs.
    - Comfy dtype: SAMPLER
    - Python dtype: T
- sigmas
    - The sigmas parameter is used to control the noise levels applied during the sampling process, directly affecting the smoothness and coherence of style-aligned samples.
    - Comfy dtype: SIGMAS
    - Python dtype: T
- ref_latents
    - The ref_latents provide a set of latent representations that serve as style alignment references, ensuring that output samples follow the desired style details.
    - Comfy dtype: STEP_LATENTS
    - Python dtype: T

# Output types
- output
    - The output parameter contains the style-aligned latent samples generated by the model, which is the primary result of the sampling process and can be further processed or analyzed.
    - Comfy dtype: LATENT
    - Python dtype: dict
- denoised_output
    - The denoised_output parameter provides refined latent samples that have been processed to reduce noise and enhance clarity, offering a clearer representation of the target style.
    - Comfy dtype: LATENT
    - Python dtype: dict
