
# Documentation
- Class name: GenerateReply
- Category: SALT/Language Toolkit/Agents
- Output node: False

The GenerateReply node generates replies between conversational agents. It produces replies based on input messages and conversation context, abstracting the complexity of conversational agent interactions to enable dynamic and responsive dialogue creation.

# Input types
## Required
- recipient
    - Specifies the agent expected to receive the message, playing a key role in determining the context and content of the generated reply.
    - Comfy dtype: AGENT
    - Python dtype: ConversableAgent
- message
    - The input message that the receiving agent is expected to reply to. This message sets the context for reply generation.
    - Comfy dtype: STRING
    - Python dtype: str

## Optional
- sender
    - Optional parameter used to identify the message sender, helping to more accurately tailor the reply within the conversation context.
    - Comfy dtype: AGENT
    - Python dtype: ConversableAgent or None

# Output types
- reply
    - The reply generated by the receiving agent in response to the input message, reflecting the agent's responsiveness in an ongoing conversation.
    - Comfy dtype: STRING
    - Python dtype: str


## Usage tips
- Infra type: `CPU`
- Common nodes: unknown

class GenerateReply:
    @classmethod
    def INPUT_TYPES(cls):
        return {
            "required": {
                "recipient": ("AGENT",),
                "message": ("STRING", {"multiline": True, "dynamicPrompts": False}),
            },
            "optional": {
                "sender": ("AGENT", {"default": None}),
            }
        }

    RETURN_TYPES = ("STRING",)
    RETURN_NAMES = ("reply",)

    FUNCTION = "start_chat"
    CATEGORY = f"{MENU_NAME}/{SUB_MENU_NAME}/Agents"

    def start_chat(self, recipient, message, sender=None):
        recipient = clone_conversable_agent(recipient)
        if sender:
            sender = clone_conversable_agent(sender)
        message = recipient._oai_messages[sender] + [{"content": message, "role": "user"}]
        reply = recipient.generate_reply(message, sender)
        return (reply,)
