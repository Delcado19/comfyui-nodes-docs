# Documentation
- Class name: ModelMergeBlockNumber
- Category: model_merging
- Output node: False
- Repo Ref: https://github.com/comfyanonymous/ComfyUI_experiments

The ModelMergeBlockNumber node merges multiple models into a single, cohesive structure. It achieves this by merging specified blocks from two input models, ensuring the resulting model retains the functionality and performance of its components. This node plays a key role in creating unified models that leverage the strengths of their parts.

# Input types
## Required
- model1
    - The 'model1' parameter is the first model to merge. It plays a critical role in determining the architecture and functionality of the final merged model. This parameter is essential as it significantly contributes to the overall structure and performance of the output model.
    - Comfy dtype: MODEL
    - Python dtype: Any
- model2
    - The 'model2' parameter represents the second model to merge. Like 'model1', it is critical to the final output, influencing the characteristics and performance of the merged model. The combination of 'model1' and 'model2' aims to create a more powerful and versatile model than any individual one.
    - Comfy dtype: MODEL
    - Python dtype: Any
## Optional
- time_embed.
    - The 'time_embed.' parameter embeds time-related information into the model. It is especially important for models requiring temporal context. This parameter allows the model to account for time-based features, enhancing its predictive capabilities.
    - Comfy dtype: FLOAT
    - Python dtype: float
- input_blocks
    - Each 'input_blocks.i.' parameter, where 'i' ranges from 0 to 11, represents a block in the model's input layer. These blocks are critical for processing input data and preparing it for further model stages. The configuration of these blocks can significantly affect the model's ability to learn and generalize from input data.
    - Comfy dtype: COMBO[FLOAT]
    - Python dtype: List[float]
- middle_block
    - The 'middle_block.i.' parameter, for 'i' ranging from 0 to 2, corresponds to blocks in the model's middle layer. These blocks are essential for the model's intermediate processing steps, influencing how the model transforms and utilizes input data before the final output.
    - Comfy dtype: COMBO[FLOAT]
    - Python dtype: List[float]
- output_blocks
    - The 'output_blocks.i.' parameter, where 'i' ranges from 0 to 11, defines the configuration of output layer blocks. These blocks are responsible for converting the model's internal representations into the desired output format. Their settings are critical for the model to produce accurate and meaningful results.
    - Comfy dtype: COMBO[FLOAT]
    - Python dtype: List[float]
- out.
    - The 'out.' parameter specifies the final output configuration of the model. It is a critical component that determines the structure of the model's predictions or outputs. Properly setting this parameter ensures the model's output aligns with the expected format and requirements.
    - Comfy dtype: FLOAT
    - Python dtype: float
- label_emb.
    - The 'label_emb.' parameter embeds label information into the model. This is particularly useful when the model needs to understand and incorporate categorical data or class labels into its processing. Embedding enhances the model's ability to make informed decisions based on label data.
    - Comfy dtype: FLOAT
    - Python dtype: float

# Output types
- merged_model
    - The 'merged_model' output represents the combined model generated by the merge process. It encapsulates the integrated characteristics and functionalities of the input models, providing a more robust and versatile solution for prediction tasks.
    - Comfy dtype: MODEL
    - Python dtype: Any
